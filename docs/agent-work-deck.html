Slide 4

Defect Found During QA

The browser run exposed a real integration bug that static tests did not catch.

Symptom

  • Local demo tree loaded, but the app status reported a failure.
  • The message was demo load failed: _kfSetPeopleControlsCollapsed is not defined.
  • The failure was triggered by selected-person card code calling a stale helper after the refactor.

Fix

  • Restored _kfSetPeopleControlsCollapsed(collapsed) in src/app/49-ux-people-panel.js.
  • The helper updates shared UX state and rerenders the People controls.
  • Added test/ui-refactor-contract.test.mjs so this specific contract fails in CI if removed again.

Slide 7

Operational Notes For Next Agent

Use this to avoid repeating known setup mistakes.

Local Demo Setup

pnpm run seed-demo:local -- "Golden - Rosenberg.normalized.ged" gazetteer.json golden-rosenberg pnpm run dev pnpm run smoke:demo:local -- http://127.0.0.1:8791/api/demo

Do not add --r2 STORAGE to local Pages dev. That creates an isolated binding-named local bucket instead of the configured genealogy-flow bucket.

Port Notes

  • Port 8788 was occupied by an unrelated local service during QA.
  • The verified app was run on 8791.
  • The local server was stopped after testing.
  • Untracked GEDCOM/data files are private artifacts and should not be deleted or committed.

Slide 9

Agent Handoff Checklist

Minimal steps before the next push touching the UI.

Before Editing

  • Run git status --short and preserve untracked private data files.
  • Read the relevant src/app/NN-*.js fragment, not only generated public/app.js.
  • Regenerate public assets with pnpm run predeploy before browser testing.

Before Closing

  • Run pnpm run test:ux, typechecks, and build.
  • Smoke local demo API against the same port the app uses.
  • Use browser/CDP for at least one desktop and one mobile interaction path.
  • Commit only source/docs/tests, not local GEDCOM artifacts.
